2) Что такое git push 

Это команда гит, которая отправляет закомиченные изменения с указанной в команде ветки на удаленный репозиторий.

3) Git merge и rebase (что делают, в чем отличие)

Обе команды используются для слияния веток. Отличие в том, что при использовании команды git merge
просто создаётся своего рода новый комит на основной ветке, тем самым сохраняются все коммиты на старой ветке и сама ветка. При использовании
Git rebase все изменения со старой ветки консолидируются и накладываются на основную, при этом переписывается вся история проекта.

принцип ООП озночает, что методы класса не доступны из вне и нас не касаются детали их реализации и то как он работает внутри, а также сокрытие полного

9) Что такое json

Текстовый тип данных, представляющий из себя пары ключ-значения со вложенностью. Является популярным форматом, так многие языки программирование имеют 
встроенные средства, для работы с ним и позволяет взаимодействовать приложениям написанным на разных языках. Также используется для передачи данных в вебе.
В python тип данных dict соответвует формату json.

11) Что такое многие-ко-многим, пример связи

Для построения связи многие-ко-многим должна быть создана промежуточная таблица в которой построены связи один-ко-многим с двумя или более другими таблицами. 
Например есть таблица содержащая название дисциплин в университете и таблица содержащая список учебных групп. Так одна дисциплина может изучаться множеством
групп и одна группа может изучать множество дисциплин создаётся промежуточная таблица для построения связи многие-ко-многим между
группами и дисциплинами, например для создания расписания занятий. 

12) Что такое JOIN и какой он может быть

JOIN это оператор в языке sql для объединение таблиц. 

Виды:

inner(natural) join - таблицы объединяются при наличии одинаковых названий столбцов (либо указать с помощью JOIN ON по каким столбцам проводить объединение),
 не соответствующие критериям строки объединены и выведены не будут.

outer join - 
left, right  будет выведены строки которые не соответвуют критериями объединения из левой или правой таблицы соответственно
full - будет выведены строки которые не соответвуют критериями объединения из обеих таблиц
Для строк которые не соответвуют критериям в новых столбцах будет отсутствовать значение (null)

cross join - все строки первой таблицы объединяются со всеми строками второй таблицы

self join - таблица объединяется со своей копией, если нужно повторно вывести данные которые уже содержаться в таблице.
